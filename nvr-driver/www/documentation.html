Frigate NVR Controller for Control4

Auto-discovers cameras from Frigate NVR, creates Control4 camera drivers, and delivers real-time detection events via MQTT.

Quick Start

  1. Ensure frigate-camera.c4z is loaded on the controller (System Design → Manage Drivers)
  2. Add the Frigate NVR driver to your project
  3. Set Frigate Host and MQTT Broker addresses
  4. Verify Frigate Status shows "Online" and MQTT Status shows "Connected"
  5. Run the Discover Cameras action — camera drivers appear automatically
  6. Move each camera driver to the appropriate room
  7. Optionally run Rename Cameras to Frigate Names for clean display names
That's it! Detection events, history, and Composer variables start flowing automatically. Each camera records events to its own history timeline.

Actions

ActionDescription
Discover Cameras Queries Frigate API for all cameras. Adds a camera driver for each new camera. Existing cameras are updated with current host/sub-stream settings. Safe to run multiple times.
Rename Cameras to Frigate Names Renames all managed camera drivers to match Frigate names (title-cased, underscores to spaces). e.g. driveway_to_gate → Driveway To Gate. Refreshes the Composer project tree.
Reconnect MQTT Disconnects and reconnects the MQTT client. Use if events stop flowing.
Check Frigate Status Tests Frigate API connectivity and reports the Frigate version.
Remove All Cameras Clears the NVR driver's camera tracking. Camera drivers must be removed manually in Composer.

MQTT Event Routing

The NVR driver subscribes to Frigate's MQTT topics and routes events to the appropriate camera driver.

Subscribed Topics

TopicPurpose
frigate/availableFrigate online/offline → all cameras
frigate/+/personPerson count per camera
frigate/+/carCar count per camera
frigate/+/dogDog count per camera
frigate/+/catCat count per camera
frigate/+/motionMotion ON/OFF per camera
frigate/+/+/+Zone events (camera/zone/object)
frigate/eventsFull event JSON (for loitering detection)

Event Flow

Camera → Frigate detects object → MQTT broker
    ↓
NVR Driver (parses topic, determines camera + event type)
    ↓
Camera Driver (fires C4 events, updates variables, records history)
    ↓
Composer programming triggers (lights, locks, scenes, notifications)

How It Works

  • Each camera is an independent driver instance — placeable in any room
  • The NVR driver tracks cameras persistently (survives reboots)
  • On startup, stale tracking entries are cleaned up automatically
  • MQTT auto-reconnects on disconnect (30-second interval)
  • Camera drivers work standalone for streaming even without the NVR driver

Requirements

  • Control4 OS 3.2.0 or newer (required for C4:AddDevice API)
  • Frigate NVR 0.14+ with go2rtc and MQTT enabled
  • MQTT broker accessible from the Control4 controller
  • frigate-camera.c4z loaded on the controller before running Discover

Troubleshooting

ProblemSolution
Frigate Status: Offline Check network connectivity. Verify Frigate API at http://host:5000/api/version
MQTT Status: Disconnected Verify broker is running. Check IP and port. Try Reconnect MQTT action.
Discover finds 0 cameras Verify http://host:5000/api/config returns camera data in a browser
Cameras not receiving events Check MQTT Status is "Connected". Verify Managed Cameras lists the camera name.
Camera added but no video Camera may not be configured yet. Check its Camera Name and Frigate Host properties.
