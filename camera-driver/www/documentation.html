Frigate Camera Driver for Control4

Displays a Frigate NVR camera in Control4 navigators with full detection event support.

Setup

This driver can be added manually or auto-created by the Frigate NVR parent driver.

Automatic (recommended)

  1. Add the Frigate NVR driver first
  2. Run Discover Cameras — this driver is created automatically for each camera
  3. Move the camera to the appropriate room
  4. Optionally run Rename Cameras to Frigate Names from the NVR driver

Manual

  1. Add this driver in Composer Pro
  2. Set Frigate Host to your Frigate server IP address
  3. Set Camera Name to the exact Frigate camera name (e.g. front_door)
  4. Set Use Sub Stream to Yes for H.264 720p (recommended) or No for main stream

Stream Types

TypePortUsed ByURL Pattern
MJPEG1984Touchscreens, On-Screen, PC http://host:1984/api/stream.mjpeg?src=camera_sub
RTSP H.2648554Mobile Apps rtsp://host:8554/camera_sub
Snapshot5000Thumbnails http://host:5000/api/camera/latest.jpg
Sub-streams recommended: Control4 navigators cap at 720p and H.264 sub-streams avoid transcoding. Main streams use H.265 which Control4 cannot decode.

Detection Events

When paired with the Frigate NVR driver, this camera receives real-time detection events via MQTT. Events appear in the Control4 app history and are available for Composer programming.

History

All detection events are recorded in the device history, visible in the Control4 app timeline and on touchscreens. Loitering events are recorded with Warning severity; all others are Info.

Requirements

  • Control4 OS 3.2.0 or newer
  • Frigate NVR 0.14+ with go2rtc enabled
  • go2rtc RTSP restream on port 8554, HTTP API on port 1984
  • Frigate REST API on port 5000
  • For detection events: Frigate NVR parent driver + MQTT broker

Troubleshooting

ProblemSolution
No video on touchscreen Verify MJPEG URL in browser: http://host:1984/api/stream.mjpeg?src=camera_sub
No video in mobile app RTSP may 404 if go2rtc has no active producer. MJPEG should still work.
No snapshot Verify: http://host:5000/api/camera/latest.jpg
No detection events Ensure Frigate NVR driver is added, MQTT Status shows "Connected", and MQTT broker is running
Events not in history Check that the NVR driver's Managed Cameras property lists this camera's name
